## 💡 经验教训

### 1. CI/CD 自动化优先
**当 CI/CD 自动化到位后，本地发布脚本的价值降低。**

- GitHub Actions 自动构建流程稳定可靠
- 本地脚本（`scripts/release.sh`）可降级为测试用途
- 优先投资于自动化 CI/CD 流程

### 2. 务实决策
**根据实际情况调整方案，而非死守原计划。**

- 发现 Gitee 容器镜像下线 → 调整为阿里云 ACR
- GitHub Actions 稳定可靠 → 作为主要构建方式
- 双备份策略 → GitHub 主仓库 + Gitee 镜像
