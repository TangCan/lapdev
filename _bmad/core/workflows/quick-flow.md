# Quick Flow Workflow

## Overview
Quick Flow is the default workflow for rapid development cycles. It follows the BMAD methodology with simplified steps.

## Phases

### 1. Brief
- Define requirements
- Create user stories
- Set acceptance criteria

### 2. Map
- Analyze user journeys
- Identify triggers and scenarios
- Design UX flows

### 3. Architect
- Define technical architecture
- Create system diagrams
- Plan implementation

### 4. Develop
- Implement features
- Write tests
- Code review

## Actions
- `bmad quick`: Start quick flow
- `bmad status`: Check current status
- `bmad next`: Move to next phase
