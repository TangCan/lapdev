## 1. 项目背景与目标

### 1.1 项目背景
团队长期使用 Cursor IDE，但受限于其 Token 用量上限与订阅模式。为获得完全自主可控的开发环境，决定从零构建一个支持“自带密钥”（BYOK）的纯开源 Web IDE。同时，团队习惯 BMAD 敏捷开发方法，需要将此工作流深度集成至 IDE 中。

### 1.2 项目定位
**Lapdev** 是一个完全开源（MIT）、免费、高性能的 Web IDE，运行在 Linux 服务器上，通过浏览器访问。它不基于 VS Code，拥有独立技术栈，并原生支持第三方大语言模型及敏捷开发工作流。

### 1.3 核心目标
- 提供可替代商业 IDE 的现代 Web 开发体验。
- 让用户通过自带 API 密钥自由选择大语言模型。
- 内置强大的 Skill 系统与 BMAD 方法支持，提升 AI 辅助开发的效率与规范性。
- 保证代码完全公开，零付费功能墙，并适配国内网络环境。

## 2. 技术约束

| 约束项 | 具体要求 |
| :--- | :--- |
| **后端开发语言** | 核心高性能模块使用 **Rust**；业务逻辑与 API 服务使用 **Deno/TypeScript** |
| **前端开发语言** | TypeScript + React |
| **运行环境** | Linux（开发与生产），通过容器镜像分发 |
| **用户界面** | 基于 Web 浏览器，非桌面原生应用 |
| **开源协议** | MIT |
| **AI 模型接入** | 支持“自带密钥”（BYOK），可配置 API Key 与 Base URL |
| **编辑器内核** | Monaco Editor（独立组件，非 VS Code 平台） |
| **终端模拟** | xterm.js + Rust PTY |

## 3. 功能需求

### 3.1 核心 IDE 功能
- **文件管理**
  - 文件树显示与实时更新（基于 inotify）
  - 文件 / 文件夹的创建、重命名、删除
  - 支持 .gitignore 忽略规则
  - 多文件 Tab 打开与切换
- **代码编辑器**
  - 语法高亮、自动括号匹配、代码折叠
  - 多光标编辑
  - 主题切换（浅色 / 深色）
- **终端**
  - 支持多个终端 Tab
  - 完整的 Shell 环境（bash / zsh 等）
  - 可调整终端尺寸
- **Git 集成**
  - 显示 Git 状态（修改、暂存、未跟踪）
  - 可视化 diff 对比
  - 支持 stage、commit、branch 切换等基础操作
- **LSP 代码智能**（第一阶段至少支持 Rust 和 TypeScript）
  - 自动补全
  - 悬停提示
  - 跳转到定义
  - 查找引用
  - 实时诊断（错误 / 警告波浪线）

### 3.2 AI 功能（BYOK）
- **AI 聊天面板**
  - 侧边栏对话窗口
  - 支持引用当前打开文件或选中代码作为上下文
  - 流式输出模型回复
- **内联代码补全**
  - 在编辑器中触发 AI 补全建议
  - 幽灵文本显示，Tab 键接受
- **Agent 模式**
  - 允许 AI 在受限权限下读取文件、搜索代码库、生成并应用修改
  - 所有影响文件的操作必须经过用户确认
- **模型配置面板**
  - API Key 输入（加密存储于服务端会话或本地）
  - Base URL 自定义（兼容 OpenAI、DeepSeek 等）
  - 模型名称选择
  - 可配置多个模型并快速切换

### 3.3 Skill 系统
- **Skill 文件规范**
  - 每个 Skill 由 `.skill.md` 文件定义
  - 包含元数据（名称、描述、版本）及结构化指令
- **Skill 加载**
  - 自动扫描全局目录 `~/.lapdev/skills/` 和项目级 `.lapdev/skills/`
  - 启动时动态加载，无需重启
- **Skill 市场**
  - 提供 CLI 命令：`lapdev skill install`、`lapdev skill publish`
  - 官方维护 `skills.lapdev.dev` 社区市场（可通过 Gitee Pages 等服务部署）
- **Skill 运行时**
  - AI Agent 根据用户请求自动匹配并激活相关 Skill
  - 支持多个 Skill 协同工作

### 3.4 BMAD 敏捷方法论支持
- **默认内置**
  - BMAD 作为 Lapdev 的一等公民技能套件，开箱即用
- **安装策略**
  - 优先执行 `npx bmad-method install` 在线安装完整版
  - 如不可达，自动降级至内置简化版核心文件
- **简化版核心文件（Fallback）**
  - 包含 `_bmad/core/config.yaml`
  - `_bmad/core/workflows/quick-flow.md`
  - `_bmad/core/agents/developer.md`
  - `_bmad/core/agents/pm.md`
- **降级安装流程**
  - 检测 Node.js 环境
  - 若无 Node.js 或在线安装失败，直接将内置文件写入项目
  - 自动在 Lapdev 的 Skill 注册表中注册 BMAD 相关技能
- **Docker 环境保障**
  - Lapdev 官方容器镜像基于 `node:20-alpine`，内置 Node.js、npm、npx
  - 确保正常网络下可直接在线安装完整 BMAD

## 4. 架构概要

### 4.1 前后端分层
```
Browser (Monaco, xterm.js, React UI)
        │ WebSocket / HTTP
Deno API Gateway (路由, 会话, WebSocket管理)
        │ FFI / 子进程
Rust Core Services (文件服务, LSP管理, 终端PTY, AI代理)
```

### 4.2 主要模块
- **lapdev-web**：前端 React 应用
- **lapdev-server**：Deno/TypeScript 后端服务
- **lapdev-core**：Rust 核心库（提供高性能操作）

### 4.3 通信机制
- 编辑器同步、LSP 消息、终端 I/O 均通过 WebSocket 全双工通信
- AI 请求通过 Rust 侧代理，统一附加 Token 后转发至用户指定的 LLM API

## 5. 容器化与部署

### 5.1 容器引擎：Podman（平替 Docker）
- Lapdev 全面兼容 Podman，官方文档中将其作为主要推荐方案
- 项目根目录提供 `podman-compose.yml` 以方便一键启动
- 编写 `scripts/setup_podman.sh` 用于在目标 Linux 环境自动化安装和配置 Podman

### 5.2 国内镜像加速
- 在文档中提供详细指引，指导用户配置国内镜像源（如阿里云、中科大），加速基础镜像拉取
- 官方镜像推送到国内 registry（如阿里云 ACR），并提供离线镜像包供下载（通过 Gitee Releases 分发）
- 私有部署场景下，提供搭建内部 Harbor registry 并推送 Lapdev 镜像的完整文档

### 5.3 分发方式
- **在线安装**：从国内 registry 直接拉取
- **离线安装**：下载 `lapdev.tar` 镜像包，通过 `podman load` 导入
- **源码构建**：提供脚本利用 CNB 等国内平台的自动构建服务，实现从源码到镜像的一键构建

## 6. 代码托管与协作（国内优先）

### 6.1 代码仓库
- **主仓库**：**Gitee (码云)**，作为国内开发者的主要访问和协作入口
- **镜像仓库**：**GitHub**，通过 Gitee 的自动镜像功能保持同步，确保国际可见度

### 6.2 社区协作
- Issue 和 PR 以 Gitee 为唯一官方入口，GitHub 侧置顶说明同步策略
- 通过 Gitee DevOps 配置 CI/CD 流水线，实现自动化构建、测试和部署

## 7. 非功能性需求

### 7.1 性能
- 服务器启动时间 < 2 秒
- 大型文件（>10k 行）打开延迟 < 500ms
- 终端输入到显示延迟 < 50ms

### 7.2 安全性
- 工作区文件访问严格限制在指定根目录内
- 用户 API Key 仅存储于服务端内存，不持久化，不打印日志
- Agent 执行的所有文件操作需用户明确授权
- 未来插件在 Deno 沙箱中运行，遵循最小权限原则

### 7.3 可部署性
- 提供单容器 Podman 镜像，通过环境变量配置工作区路径
- 支持通过 `podman-compose` 快速部署

### 7.4 可维护性
- Rust 与 Deno 间通过 FFI 交互，接口需定义清晰的 C ABI
- Deno 端使用 TypeScript 提供类型安全
- 前后端约定清晰的 WebSocket 消息协议（JSON 格式）

### 7.5 文档与社区
- 提供完整的构建指南
- 提供 Skill 开发规范文档
- 采用开放治理，欢迎社区贡献

## 8. 开发路线图

| 阶段 | 时间 | 目标 |
| :--- | :--- | :--- |
| **Phase 1** | 8-12 周 | 基础骨架：文件树、Monaco 编辑器、Rust PTY 终端、Deno 服务、Podman 支持 |
| **Phase 2** | 6-8 周 | LSP 智能代码支持（Rust/TS）、Git 集成 |
| **Phase 3** | 4-6 周 | AI 聊天、内联补全、模型配置、Skill 系统框架 |
| **Phase 4** | 3-4 周 | BMAD 完整集成（含降级策略）、Skill 市场雏形、国内镜像分发 |
| **Phase 5** | 持续 | Agent 模式增强、协作编辑、性能调优、文档完善 |

## 9. 验收标准

1. 在无网络环境下，通过 Podman 启动 Lapdev，新建项目后可自动启用内置 BMAD 工作流。
2. 在有网络环境下，打开任意项目，执行 `npx bmad-method install` 成功后可无缝切换到完整 BMAD。
3. 用户可在设置中填入自己的 OpenAI / DeepSeek API Key 并正常进行对话、补全。
4. 文件树、编辑器、终端、Git 面板基础功能可用且无明显 Bug。
5. 所有代码在 Gitee 公开，遵循 MIT 协议，并自动同步至 GitHub 镜像。
